# 产品需求文档（PRD）：AI模拟炒股大师

## 1. 产品概述
- **产品名称**：AI模拟炒股大师
- **版本**：v1.0 MVP
- **一句话描述**：AI驱动的智能模拟炒股平台，零成本练手、AI实时指导、策略验证、社区竞技
- **背景**：基于调研发现，现有模拟炒股平台缺乏AI辅助和策略验证能力，新手学习成本高
- **目标**：上线3个月内获取10万注册用户，日活1万

## 2. 目标用户

| 用户类型 | 特征 | 核心需求 | 使用频率 |
|---------|------|---------|---------|
| 投资小白 | 18-25岁，学生/职场新人，零经验 | 学习基础、零风险练手、AI指导 | 每日 |
| 进阶玩家 | 25-35岁，有基础但想验证策略 | 策略回测、数据分析、优化建议 | 每周3-5次 |
| 竞技达人 | 全年龄段，喜欢挑战和排名 | 排行榜、PK赛、成就系统 | 每日 |

## 3. 核心功能

### 3.1 功能总览
| 功能模块 | 功能点 | 优先级 | 状态 |
|---------|--------|--------|------|
| 账户系统 | 注册/登录/虚拟资金 | P0 | 待开发 |
| 行情中心 | 实时行情/K线/自选股 | P0 | 待开发 |
| 模拟交易 | 买入/卖出/撤单/持仓 | P0 | 待开发 |

| 策略回测 | 自定义策略/历史回测/报告 | P1 | 待开发 |
| 社区竞技 | 排行榜/挑战赛/分享 | P1 | 待开发 |
| 学习中心 | 投资课程/术语词典/每日一题 | P2 | 待开发 |

### 3.2 功能详细说明

#### 3.2.1 虚拟账户系统
- **用户故事**：作为投资小白，我希望获得虚拟资金进行模拟交易，以便零风险学习投资
- **前置条件**：用户完成注册
- **操作流程**：
  1. 注册后自动获得100万虚拟资金
  2. 可在个人中心查看资产总览
  3. 资金变动实时更新
- **后置结果**：账户可用资金减少/增加，持仓更新
- **异常处理**：
  - 资金不足 → 提示"可用资金不足，无法买入"
  - 网络异常 → 本地缓存，恢复后同步

#### 3.2.2 实时行情与交易
- **用户故事**：作为投资者，我希望查看实时行情并执行模拟交易
- **操作流程**：
  1. 搜索或浏览股票列表
  2. 查看个股详情（K线、财务数据、新闻）
  3. 输入买入/卖出数量和价格
  4. 确认交易
- **交易规则**：
  - A股规则：T+1，涨跌幅限制±10%（ST±5%）
  - 交易时间：9:30-11:30, 13:00-15:00（模拟盘可24小时练习）
  - 手续费：佣金万分之3，最低5元；印花税千分之1（卖出）
- **异常处理**：
  - 非交易时间 → 提示"当前非交易时间，可预约下单"
  - 涨停/跌停 → 提示"该股票已涨停/跌停，无法买入/卖出"

#### 3.2.3 AI智能助手（核心差异化）
- **用户故事**：作为新手，我希望获得AI投资建议，以便做出更明智的决策
- **功能点**：
  1. **持仓诊断**：分析当前持仓，给出评分和优化建议
  2. **买卖时机**：基于技术指标给出买入/卖出信号
  3. **风险提醒**：持仓过于集中、个股风险过高时预警
  4. **智能问答**：自然语言询问股票相关问题
- **AI回答示例**：
  - 用户问："我现在持仓怎么样？"
  - AI答："您的当前持仓评分为72分。优点：分散度较好，涵盖3个行业。风险：科技股占比过高（65%），建议适当配置防御性板块。个股建议：XX股票近期RSI超买，可考虑减仓。"
- **免责声明**：所有AI建议仅供参考，不构成投资建议

#### 3.2.4 策略回测
- **用户故事**：作为进阶玩家，我希望验证我的投资策略在历史数据中的表现
- **操作流程**：
  1. 选择策略模板（均线交叉、MACD、自定义）
  2. 设置参数（股票池、时间范围、初始资金）
  3. 运行回测
  4. 查看报告（收益率、最大回撤、夏普比率等）
- **回测指标**：
  - 总收益率、年化收益率
  - 最大回撤、回撤天数
  - 夏普比率、胜率
  - 交易次数、盈亏比

#### 3.2.5 社区竞技
- **用户故事**：作为竞技达人，我希望与其他用户PK，展示我的投资能力
- **功能点**：
  1. **收益率排行榜**：日榜/周榜/月榜/总榜
  2. **挑战赛**：官方定期举办主题赛（如"科技股专题赛"）
  3. **组合分享**：分享持仓，其他用户可"抄作业"
  4. **讨论区**：个股讨论、策略交流

### 3.3 界面设计

#### 3.3.1 页面结构
```
首页（Dashboard）
├── 资产总览卡片（总资产/盈亏/收益率）
├── 大盘指数（上证/深证/创业板）
├── 自选股列表
└── AI每日建议

行情页
├── 股票列表（涨跌排行/行业分类）
├── 搜索框
└── 个股详情（K线/分时/财务/新闻）

交易页
├── 买入/卖出切换
├── 股票代码输入
├── 价格/数量输入
└── 交易确认

持仓页
├── 持仓列表（股票/成本/现价/盈亏）
├── 盈亏分析图表
└── AI持仓诊断

策略页
├── 策略库（我的策略/系统策略）
├── 回测配置
└── 回测报告

社区页
├── 排行榜
├── 挑战赛
└── 讨论区
```

#### 3.3.2 全局交互规范
- **加载状态**：骨架屏 + 脉冲动画
- **空状态**：插画 + 引导文字 + 操作按钮
- **错误提示**：Toast通知，3秒自动消失
- **成功反馈**：绿色Toast + 轻微震动（移动端）
- **数据更新**：行情数据每3秒刷新，持仓数据实时更新

## 4. 技术栈建议
| 层级 | 技术选型 | 理由 |
|------|---------|------|
| 前端 | React 18 + TypeScript + Vite | 生态成熟、开发效率高 |
| UI库 | Ant Design + 自定义主题 | 组件丰富、可定制金融风格 |
| 图表 | ECharts / TradingView Chart | 专业金融图表 |
| 状态管理 | Zustand | 轻量、易用 |
| 后端 | Node.js + Express | 轻量、JavaScript统一 |
| 数据库 | PostgreSQL + Redis | 关系型数据 + 缓存 |
| AI服务 | Python + FastAPI + OpenAI API | AI模型调用 |
| 行情数据 | 聚宽/同花顺API | 国内A股数据 |
| 部署 | Docker + AWS/阿里云 | 容器化、弹性扩展 |

## 5. 代码风格与架构
- **代码规范**：ESLint + Prettier + Husky
- **架构模式**：前端组件化 + 后端MVC
- **命名规范**：
  - 组件：PascalCase（如：StockChart）
  - 函数：camelCase（如：getStockPrice）
  - API端点：kebab-case（如：/api/v1/stock-price）
- **文件组织**：按功能模块分文件夹

## 6. 限制条件与边界场景

### 6.1 性能要求
- 页面首屏加载 < 2秒
- 行情数据延迟 < 3秒
- 支持同时在线用户 5000人

### 6.2 安全要求
- 用户密码bcrypt加密
- API限流：每IP每分钟100次
- SQL注入防护（使用ORM参数化查询）

### 6.3 边界场景
| 场景 | 预期行为 |
|------|---------|
| 网络断开 | 显示"网络异常"提示，本地缓存数据，恢复后同步 |
| 股票停牌 | 显示"停牌"标识，禁止交易 |
| 除权除息 | 自动调整持仓成本和价格 |
| 并发交易 | 使用数据库事务，防止超卖 |
| 数据延迟 | 显示"数据延迟15分钟"提示 |

## 7. 数据埋点
