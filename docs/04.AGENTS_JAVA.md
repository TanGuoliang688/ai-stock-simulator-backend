## 2. 开发规范

### 2.1 Java代码风格
- **JDK版本**：Java 21（使用虚拟线程、新特性）
- **编码**：UTF-8
- **缩进**：4个空格
- **括号**：K&R风格（左括号不换行）
- **命名规范**：
  - 类名：PascalCase（如：TradeServiceImpl、OrderRequest）
  - 方法/变量：camelCase（如：getStockPrice、handleSubmit）
  - 常量：UPPER_SNAKE_CASE（如：MAX_ORDER_AMOUNT、DEFAULT_BALANCE）
  - 包名：全小写，反向域名（如：com.stocksim.service）
  - 数据库字段：snake_case（如：virtual_balance、created_at）
  - 接口名：以Service结尾（如：TradeService），实现类以Impl结尾

### 2.3 分层规范
```
Controller层
├── 职责：接收请求、参数校验、调用Service、返回统一响应
├── 禁止：直接操作数据库、处理复杂业务逻辑
├── 注解：@RestController、@RequestMapping、@Validated
└── 返回：Result<T> 或 PageResult<T>

Service层
├── 职责：业务逻辑编排、事务管理
├── 禁止：直接返回entity，必须转换为DTO/VO
├── 注解：@Service、@Transactional
└── 异常：抛出BusinessException

Mapper层
├── 职责：数据访问、SQL映射
├── 禁止：写复杂SQL（用MyBatis-Plus Wrapper）
├── 继承：BaseMapper<Entity>
└── 复杂查询：写在XML中
```

```java
    private final Integer code;

    public BusinessException(String message) {
        super(message);
        this.code = 400;
    }

    public BusinessException(Integer code, String message) {
        super(message);
        this.code = code;
    }

// common/GlobalExceptionHandler.java
@RestControllerAdvice
@Slf4j
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    public Result<Void> handleBusinessException(BusinessException e) {
        log.warn("业务异常: {}", e.getMessage());
        return Result.error(e.getCode(), e.getMessage());
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public Result<Void> handleValidationException(MethodArgumentNotValidException e) {
        String message = e.getBindingResult().getFieldErrors().stream()
            .map(error -> error.getField() + ": " + error.getDefaultMessage())
            .collect(Collectors.joining(", "));
        return Result.error(400, message);
    }

    @ExceptionHandler(Exception.class)
    public Result<Void> handleException(Exception e) {
        log.error("系统异常: ", e);
        return Result.error(500, "系统繁忙，请稍后重试");
    }
}
```

```java
        UserResponse response = new UserResponse();
        BeanUtils.copyProperties(user, response);
        return response;
    }
```

```typescript
// services/api.ts - 统一Axios实例
import axios from 'axios';

const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json',
  },
});

// 请求拦截器 - 添加Token
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('access_token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// 响应拦截器 - 统一错误处理 + Token刷新
api.interceptors.response.use(
  (response) => response.data,
  async (error) => {
    const originalRequest = error.config;

    if (error.response?.status === 401 && !originalRequest._retry) {
      originalRequest._retry = true;
      try {
        const refreshToken = localStorage.getItem('refresh_token');
        const response = await axios.post('/api/v1/auth/refresh', { refreshToken });
        const { accessToken } = response.data.data;
        localStorage.setItem('access_token', accessToken);
        originalRequest.headers.Authorization = `Bearer ${accessToken}`;
        return api(originalRequest);
      } catch (refreshError) {
        localStorage.clear();
        window.location.href = '/login';
        return Promise.reject(refreshError);
      }
    }

    return Promise.reject(error);
  }
);

export default api;
```

## 4. 约束条件

### 4.1 必须遵守
- **不要修改已有文件**：`StockSimApplication.java`、数据库迁移脚本已完成，不要改动
- **分层严格**：Controller只调用Service，Service只调用Mapper，禁止跨层
- **类型安全**：所有DTO/VO/Entity必须有字段类型，禁止用Object/Map传参
- **异常处理**：所有业务异常用BusinessException，禁止try-catch后吞掉异常
- **事务管理**：涉及多表操作必须加`@Transactional(rollbackFor = Exception.class)`
- **响应统一**：所有Controller返回`Result<T>`，禁止直接返回entity或裸数据
- **日志规范**：使用SLF4J + Lombok `@Slf4j`，禁止System.out.println

### 4.2 禁止事项
- **不要引入新ORM**：使用已有的MyBatis-Plus，不要用JPA/Hibernate
- **不要修改数据库表结构**：Flyway迁移脚本已确定，不要改
- **不要改变API返回格式**：按Result包装格式返回
- **不要写原生SQL**：用MyBatis-Plus Wrapper或XML，禁止JdbcTemplate
- **不要在Controller写业务逻辑**：业务逻辑全部放到Service
- **不要返回entity给前端**：必须转换为DTO/VO

## 6. 注意事项

### 6.1 容易踩坑的地方
1. **JWT密钥配置**：`application.yml`中jwt.secret必须足够长（≥256位），生产环境用环境变量注入
2. **Spring Security放行**：登录/注册接口必须放行，否则无法获取Token
3. **MyBatis-Plus主键**：`@TableId(type = IdType.AUTO)`对应数据库自增，不要用ASSIGN_ID（雪花算法）
4. **BigDecimal比较**：不要用equals，用compareTo：`balance.compareTo(amount) >= 0`
5. **时区问题**：数据库、JVM、前端统一用UTC+8，Entity用`LocalDateTime`
6. **虚拟线程**：Java 21使用`Executors.newVirtualThreadPerTaskExecutor()`，不要手动创建Thread

### 6.2 特殊业务逻辑
- **虚拟资金**：新用户默认100万，交易后实时更新，用BigDecimal精确计算
- **交易时间**：模拟盘支持24小时交易，但需提示"当前非真实交易时间"
- **手续费**：买入佣金万分之3（最低5元），卖出佣金+印花税千分之1
- **涨跌停限制**：普通股票±10%，ST股票±5%，下单时校验

### 6.3 第三方库使用限制
- **ORM**：必须用MyBatis-Plus，不要用JPA
- **缓存**：用Spring Data Redis，不要用Jedis直接操作
- **JSON**：用Jackson（Spring Boot默认），不要用Gson/FastJSON
- **HTTP客户端**：用OpenFeign或RestTemplate，不要用OkHttp直接封装

### 7.2 正确示例：Service层（带事务和分布式锁）
```java
@Service
@RequiredArgsConstructor
@Slf4j
public class TradeServiceImpl implements TradeService {

    private final TransactionMapper transactionMapper;
    private final PositionMapper positionMapper;
    private final UserMapper userMapper;
    private final StockMapper stockMapper;
    private final StringRedisTemplate redisTemplate;

    @Override
    @Transactional(rollbackFor = Exception.class)
    public OrderResponse createOrder(Long userId, OrderRequest request) {
        // 1. 分布式锁防止并发下单
        String lockKey = "order:lock:" + userId + ":" + request.getSymbol();
        Boolean locked = redisTemplate.opsForValue()
            .setIfAbsent(lockKey, "1", Duration.ofSeconds(10));

        if (!Boolean.TRUE.equals(locked)) {
            throw new BusinessException("操作过于频繁，请稍后重试");
        }

        try {
            // 2. 校验股票是否存在
            Stock stock = stockMapper.selectOne(new LambdaQueryWrapper<Stock>()
                .eq(Stock::getSymbol, request.getSymbol()));
            if (stock == null) {
                throw new BusinessException("股票不存在");
            }

            // 3. 校验交易规则（时间、涨跌停等）
            validateTradeRules(stock, request);

            // 4. 锁定用户资金（行级锁FOR UPDATE）
            User user = userMapper.selectByIdForUpdate(userId);
            BigDecimal totalAmount = request.getPrice()
                .multiply(BigDecimal.valueOf(request.getQuantity()));

            if (request.getType() == OrderType.BUY) {
                if (user.getVirtualBalance().compareTo(totalAmount) < 0) {
                    throw new BusinessException("可用资金不足");
                }
                user.setVirtualBalance(user.getVirtualBalance().subtract(totalAmount));
                userMapper.updateById(user);
            }

            // 5. 创建订单
            Transaction transaction = new Transaction();
            transaction.setOrderNo(generateOrderNo());
            transaction.setUserId(userId);
            transaction.setStockId(stock.getId());
            transaction.setType(request.getType());
            transaction.setQuantity(request.getQuantity());
            transaction.setPrice(request.getPrice());
            transaction.setAmount(totalAmount);
            transaction.setFee(calculateFee(request));
            transaction.setStatus(OrderStatus.PENDING);
            transactionMapper.insert(transaction);

            // 6. 发送异步消息处理成交（RabbitMQ）
            // rabbitTemplate.convertAndSend("trade.exchange", "trade.routing", transaction);

            OrderResponse response = new OrderResponse();
            response.setOrderNo(transaction.getOrderNo());
            response.setStatus(transaction.getStatus());
            return response;

        } finally {
            redisTemplate.delete(lockKey);
        }
    }

    private String generateOrderNo() {
        return "ORD" + System.currentTimeMillis() + RandomUtil.randomNumbers(4);
    }

    private BigDecimal calculateFee(OrderRequest request) {
        BigDecimal amount = request.getPrice()
            .multiply(BigDecimal.valueOf(request.getQuantity()));
        BigDecimal commission = amount.multiply(new BigDecimal("0.0003"));
        return commission.max(new BigDecimal("5")); // 最低5元
    }
}
```

```xml
<!-- resources/mapper/StockMapper.xml -->
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" 
    "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="com.stocksim.mapper.StockMapper">

    <select id="selectStockList" resultType="com.stocksim.vo.StockVO">
        SELECT 
            s.id,
            s.symbol,
            s.name,
            s.market,
            s.industry,
            p.close_price as currentPrice,
            p.close_price - LAG(p.close_price) OVER (PARTITION BY s.id ORDER BY p.trade_date) as changeAmount,
            (p.close_price - LAG(p.close_price) OVER (PARTITION BY s.id ORDER BY p.trade_date)) / LAG(p.close_price) OVER (PARTITION BY s.id ORDER BY p.trade_date) * 100 as changePercent
        FROM stock s
        LEFT JOIN price_history p ON s.id = p.stock_id
        WHERE p.trade_date = (SELECT MAX(trade_date) FROM price_history WHERE stock_id = s.id)
        <if test="industry != null and industry != ''">
            AND s.industry = #{industry}
        </if>
        <if test="keyword != null and keyword != ''">
            AND (s.symbol LIKE CONCAT('%', #{keyword}, '%') OR s.name LIKE CONCAT('%', #{keyword}, '%'))
        </if>
        ORDER BY changePercent DESC
    </select>
```
